## 背景

采集的数据越多，积累的抓取脚本就越多

由此产生额外的维护工作变多

需要一个抓取平台，负责调度，抓取，监控各任务状态

同时提升抓取效率

## 总体设计

![总设计架构图][2]

该抓取平台采取并列计算的架构模式

需要一个调度节点，负责监控各节点状态，分发任务

其余的均为代理节点，通过调度节点启动，负责执行调度发来的任务
